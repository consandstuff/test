Firmware / Code

Included in this guide are two versions on how to aproach this:

  • Teensy 3.2 board

    • Difficulty: medium-hard
    • Costs: expensive alternative
    • Pros: modable, code ready to upload
    • Cons: messy work, need soldering skills

  • Arduino Leonardo controller

    • Difficulty: medium
    • Costs: cheaper alternative
    • Pros: modable, friendly code, you can use jumper wires for everything
    • Cons: messy wire work, needs some Arduino IDE skills, need some soldering skills

For both versions the box is exactly the same, the only thing that'll change is the wiring stuff at the end. Included there's code for both Arduino and Teensy controllers.

Prepare your board#

First you have to get used to working with Arduino IDE for both Teensy and Leonardo boards.

I suggest you buy the board first, upload the firmware and test the different pins using a jumperwire to understand how it works.

Basically every pin in each board has a button and for the button to send a "on" state it has to be connected to a grounded pin (GND).

So you can test the buttons by uploading the firmare, connecting it to a usb port and going to https://gamepad-tester.com/ to test each pin.

Uploading the firmware#

How to upload the code on the Teensy 3.2 PCB.#

  • Download and install ARDUINO IDE v1.8

  • Download and install Teensyduino

  • Add these lines to the C:\Program Files (x86)\Arduino\hardware\teensy\avr boards.txt file and save.

teensy31.menu.usb.sdvx=SDVX controller teensy31.menu.usb.sdvx.build.usbtype=USB_ARCADE teensy31.menu.usb.sdvx.fake_serial=teensy_gateway

Then add the 4 usb_arcade and usb_desc files inside yout core/teensy3 folder and overwirte existing files.

Alternatively download the whole set of folders from the github and paste into Program Files.

  • Open Arduino IDE v1.8
  • In the Tools menu choose Board Type: Teensy 3.2
  • In the Tools menu choose USB Type: SDVX Controller
  • On the Arduino IDE screen press the verification button with a blank file (the verification button looks like a aprooved tick)
  • It'll open a small Teensyduino screen.
  • Hit the "Open HEX file" button on the Teensyduino screen and choose the hex file provided in the github repository.
  • Manually click the hardware button that's on top of the Teensy PCB, and it'll upload.
  • After a few seconds it'll be done.

Sometimes windows won't recognize the controller as a USB device to test the buttons. For testing the buttons try https://gamepad-tester.com/
